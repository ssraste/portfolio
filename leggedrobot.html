Legged Locomotion on Granular Media

Bio-inspiration locomotion ElectroMechanical Systems Control Theory

Research Objective

Animals have complex foot structures that let them run efficiently on different terrains, stick to surfaces, and even walk on water. Researchers have been studying the biomechanics of these dynamic movements.

photo by Dorottya Ujvári on Pexels

How do these complex foot shapes help animals move better on sand?

Inspired by sand-specialist lizards, we examine how different foot morphologies affect foot–ground interactions on loose sand.

Contributions

  • We upgraded a legacy robot by upgrading hardware with newer motors, motor controllers and microcontroller.
  • Designed and fabricated lightweight aluminum mounting parts via waterjet cutting to ensure precise hardware alignment during jumping experiments.
  • Developed control code with control in joint space just like a torsional spring, implemented PD control and tuned the parameters to achieve stable jumps on acrylic beads (acts like sand).
  • Created a comprehensive documentation detailing the entire robot-building process from fabrication to assembly to testing and adding additional sensors, making it accessible to other researchers.
  • Intergrated force transducer, current sensors and string potentiometer for collecting the foot-ground interactions, jump height data and system efficiency.
  • Successfully maintained the same overall weight of the robot like the older versions while accommodating heavier hardware.
  • Currently prototyping and testing different novel foot structures, both rigid and semi-flexible feet with varying toes.

    Hardware Selection

  • T motor U8 Lite L: These BLDC motors can provide high-torque at relatively low speeds, eliminating the need for gearboxes. Their high-thermal load capacity, high torque, output power and light-weight make them efficient for rapid jumps.
    • Reference paper for selection: Direct-Drive Legged Robots
  • ODrive Pro motor controller: We used odrive pro as it delivers high peak torque, supports BLDC motors, is easy to configure, and is commercially available.
    • The main drawback is that it is a single-axis controller, so we need two odrives for each motor, which adds weight to the robot. This is largely offset by lightweight Carbon fiber body and Aluminum mounts.
  • AS5047D Encoder (SPI): It offers good resolution, SPI communication, easy to use and cost effective.
    • Several design iterations were required to fix the vertical and XY gap between the encoder and the magnet and to securely mount it, to reduce slips during jumps.

    Control Logic

  • The robot leg behaves like a Virtual Spring transitioning from soft spring to stiff during impact, to achieve jumping behavior. There isn't any physical spring attached to the robot, everything is achieved via control.
  • The system uses joint-space control, where the motor angles \(\theta_0\) and \(\theta_1\) are commanded directly. The motors behave like torsional springs, applying torque to the link according to: \[ \tau_0 = k (\theta_0 - \theta_\text{neutral}) \quad \text{(for motor 1)} \] \[ \tau = k (\theta_1 - \theta_\text{neutral}) \quad \text{(for motor 2)} \] where \(\theta_\text{neutral}\) is calculated from the vertical rod length \(l\) using the following equations: \[ l_1 = 0.1\ \text{m}, \quad l_2 = 0.2\ \text{m}, \quad l = 0.3\ \text{m}, \quad l_{\text{neutral}} = 0.27\ \text{m}, \quad l_{\text{compressed}} = 0.1\ \text{m} \]

    We derive the equation relating \( l \) and \( \gamma \) as \( l = l_1 \cos \gamma + \sqrt{l_2^2 - l_1^2 + l_1^2 \cos^2 \gamma} \)

    \[ \gamma = \cos^{-1}\!\left(\frac{l^2 + l_1^2 - l_2^2}{2 l_1 l}\right) \quad {\scriptstyle \text{(we get \(\gamma\) in radians, but ODrive only understands revolutions)}} \] We can derive motor angles \(\theta_0\) and \(\theta_1\) by: \[ \alpha = -\theta_0 + \frac{\pi}{2} \] \[ \beta = \theta_1 - \frac{\pi}{2} \] \[ 2 \gamma = \alpha + \beta \] Since the robot is symmetrical, we have: \( \alpha = \beta = \gamma \)

    We can relate the rod length \( l \) to the motor angles \( \theta_0 \) and \( \theta_1 \): at \(l\) = 0.27m, \(\theta_0\) = 0.1465 rev

  • The vertical rod can tilt on impact. Therefore, to prevent leg swinging, we constrain the tilt angle \(\delta\), which is defined as: \(\delta\) = \(\theta_0\) - \(\theta_1\) < 0.005 rev
  • Low-level control is implemented on the ODrive, while a Teensy microcontroller handles high-level stiffness switching command.
